### Supabase 还需要你额外配置什么？

如果你想真正启用云端能力，你这边只需要额外准备这些：

1. **Supabase 项目**
   - 创建一个项目
   - 拿到 `Project URL`
   - 拿到 `anon key`

2. **前端 `.env`**
   - `VITE_SUPABASE_URL=...`
   - `VITE_SUPABASE_ANON_KEY=...`
   - 如果走函数网关，还可以加：
     - `VITE_GATEWAY_PROVIDER=supabase`
     - `VITE_SUPABASE_FUNCTIONS_URL=...`（可选）

3. **数据库 migration**
   - 执行 `supabase/migrations/` 里的 SQL

4. **Edge Functions secrets**
   - `AI_BASE_URL`
   - `AI_API_KEY`
   - `AI_MODEL`
   - 可选：`TAVILY_API_KEY`

如果你**暂时不配这些**，项目仍然能跑，只是会继续走 **本地缓存 + 本地知识库兜底**。

## 1. 你指出的两个 UI / 产品问题，我认同

### 问题 A：疾控地图左边下方留白太大，不协调

这确实是当前设计问题，不是数据问题。

**根因：**
- 左侧只有一个固定高度地图卡片
- 右侧塞了很多解释卡和信源卡
- 当右侧内容更高时，左侧就会出现一大片空白

**下一步改法：**
- 把左侧改成 **“地图 + 下方辅助卡”两层结构**
- 在地图下面补：
  - 热点症状榜
  - 重点告警摘要
  - 区域快照 / 最近上报趋势

这样左边不会空，而且更像真实监测平台。

### 问题 B：首页“演示模式”太丑、太像为了比赛硬凑

这个判断我同意，应该删。

**下一步改法：**
- 删除单独的“演示模式 / 演示场景”大卡块
- 首页只保留：
  - 价值主张
  - 家庭守护模式
  - 常见症状快捷入口
  - 云端档案 / 历史记录入口

整体会轻很多，也更像真正产品首页。
