# 健康助手下一轮修复与重构计划

## 1. 先直接回答这轮你指出的问题

### 1.1 “本机记录”当前是什么意思

它现在的含义是：

- **还没有进入真实云端账号会话**
- 资料和问诊记录只保存在**当前浏览器 / 当前机器的 localStorage**
- 换设备、换浏览器、清缓存之后，这些数据就不会跟着走

所以你现在的判断是对的：  
**它还不能算真正完成的账号登录与跨设备同步体验。**

后续要把这个状态重新定义为：

- `游客模式（仅当前浏览器保存）`
- `登录后可同步`
- `已登录并云端同步`
- `同步异常，已回退本地`

并且这些状态要**可点击、有解释、有动作**，不能只是一个无意义标签。

### 1.2 疾病地图现在确实还有阻塞性 bug

从你截图看，当前地图页至少有两个问题：

1. `officialSourceHelpers.useDashboardOfficialSources is not a function`
   - 这是一个 **runtime import / hook 调用异常**
   - 会直接导致地图页白屏

2. Vite dev 的 websocket / HMR 仍然在控制台刷报错
   - 虽然不一定影响生产环境
   - 但它会放大“页面像在 debug / 没有完成”的观感

这一轮必须把这些阻塞问题先彻底收掉。

### 1.3 当前“用户中心 / 健康档案”的结构仍然不对

你指出的几个点都成立：

- 右侧大片空白，说明布局没有围绕真实使用场景设计
- “最近问诊记录”直接放在档案编辑页旁边，不像真实产品
- 档案表单太长，像后台配置页，不像用户中心
- 缺少“像 AI 产品那样的历史会话列表 + 点击继续聊”的体验

所以这一轮不能只继续堆字段，而是要**重新组织用户中心的信息架构**。

---

## 2. 这一轮的明确目标

本轮不再做泛泛的“继续优化”，而是集中完成 5 个真实问题：

1. **修掉地图和本地运行的阻塞 bug**
2. **把登录真正做成可理解、可完成、可同步的账号体系**
3. **重构健康空间 / 用户中心**
4. **把“最近问诊记录”升级成 AI 风格的历史会话**
5. **去掉所有仍然残留的 debug / 开发态暴露**

## Wave 2：把登录做“真”

### 当前问题

虽然已经接了 Supabase Magic Link，但用户感知层面仍然像“只是一个本地模式 + 隐藏的登录入口”。

### 本轮目标

做成一个明确的账号体系：

- 游客直接用
- 登录后自动同步
- 登录状态一眼能看懂
- 同步失败会解释原因

### 计划

1. 把 Header 上的状态从“本机记录”改成真正可理解的分层：
   - 游客模式（仅当前浏览器保存）
   - 点击可查看说明
   - 可直接跳转到登录 / 同步面板

2. 强化 Magic Link 流程：
   - 发送中、已发送、等待验证、验证成功
   - 回跳后自动识别登录成功并提示
   - 如果 Supabase 会话失败，直接显示可理解的原因，而不是笼统的“当前改用本机记录”

3. 区分“是否真的云端同步成功”
   - 不是只显示一个 badge
   - 要展示：
     - 当前邮箱
     - 最近同步状态
     - 当前保存位置（本地 / 云端）

### 相关文件

- `src/lib/supabase.ts`
- `src/hooks/useHealthWorkspace.ts`
- `src/components/Header.tsx`
- `src/components/CloudSyncCard.tsx`
- `src/App.tsx`

## Wave 3：重构健康空间 / 用户中心

### 当前问题

现在的健康空间更像“资料填写页”，不是“用户中心”。

### 新结构（本轮推荐直接改）

把用户中心拆成三个明确区域：

#### 1. 账号概览

展示：

- 当前模式（游客 / 已登录 / 同步异常）
- 登录邮箱
- 最近同步时间
- 云端 / 本地记录条数

#### 2. 健康档案

只保留必要信息：

- 昵称
- 年龄 / 性别
- 常驻城市
- 慢病 / 过敏 / 当前长期用药
- 关注重点

并压缩成更紧凑的卡片式布局，避免出现你截图里的大面积空白。

#### 3. 问诊历史

不再只是“最近几条摘要”，而是做成 **历史会话列表**：

- 每条会话有标题、时间、风险等级
- 可点击恢复并继续聊天
- 登录后同步到 Supabase，游客时先保存在本地

### 相关文件

- `src/components/CloudSyncCard.tsx`
- `src/App.tsx`
- `src/hooks/useHealthWorkspace.ts`
- `src/lib/healthData.ts`

## Wave 4：把问诊记录改成 AI 风格“历史会话”

### 目标

像 AI 产品一样：

- 左侧 / 上方是历史会话列表
- 点击后可以恢复上下文继续聊
- 不再把“最近问诊记录”硬塞在档案表单右边

### 计划

1. 新增 `conversation sessions` 数据结构：
   - session id
   - 标题（由首轮症状自动生成）
   - 最后更新时间
   - 当前分级
   - 消息列表

2. 每次问诊完成后自动存档：
   - 游客 → localStorage
   - 登录用户 → Supabase

3. 用户点击历史会话后：
   - 恢复 `messages`
   - 恢复诊断结果
   - 继续在原上下文上对话

### 相关文件

- `src/hooks/useChat.ts`
- `src/lib/healthData.ts`
- `src/types/index.ts`
- 新增 `src/components/ConversationHistoryPanel.tsx`
