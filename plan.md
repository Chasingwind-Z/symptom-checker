# 健康助手 Beta：信息架构重构与正式上线计划

## 一、先直接回答你这次看到的问题

### 1. 你在 Supabase 里看到的 Email 已开启，这一步是对的

从你截图看：
- **Email Provider 已经启用**
- **URL Configuration 里已经加了 localhost 和 Vercel 域名**

这两步说明 **Auth 基础配置方向是正确的**。

### 2. 为什么 Vercel 还是旧页面

根因不是 Supabase，而是 **部署分支没同步**：

- 你现在 GitHub 截图看到的是 **`main`**
- 我最近的大部分改动都在 **`feature/dynamic-suggestions-and-weather`**
- Vercel 的 **Production** 也还在指向 `main`

所以会出现：
- 本地代码已经更新很多
- feature 分支也有新提交
- 但 GitHub 首页 README 和 Vercel 生产环境仍然显示旧版本

### 3. 为什么你觉得首页丑、重、像 demo

这个判断是对的，当前页面主要有 4 个问题：

1. **信息架构混乱**：账号同步、问诊入口、疾病地图、云状态都堆在同一屏
2. **首页太“重”**：卡片太多，层级太平
3. **开发态信息露出过多**：例如“云端读取异常 / Auth session missing”这类技术词不该直接露给用户
4. **公卫信息可信感不够**：虽然底层已经有官方信源框架，但页面上仍然容易被看成“演示数据”

---

## 二、这轮重构的目标

把它从“黑客松功能堆叠页”改成更像真实产品的结构：

- **游客模式优先**：用户一打开就能直接开始症状自查
- **账号体系单独承载**：登录、档案、历史记录放到独立工作区，不压在首页
- **疾病地图 / 公卫信息独立页面**：不再和问诊主流程挤在一起
- **GitHub / README / Vercel Production 同步到最新版**
- **弱化 demo 感，强化可信感和可落地感**

---

## 三、产品结构重做方案

### 方案 A：三段式结构（本轮推荐直接实施）

#### 1. 首页 / 问诊入口

首页只保留：
- 产品价值主张
- “游客开始自查”主按钮
- “登录同步档案”次按钮
- 常见症状快捷入口
- 少量可信说明（非大段堆料）

首页不再直接塞：
- 大块云同步卡
- 历史记录卡片
- 技术错误文案
- 大片公共卫生解释模块

#### 2. 健康档案 / 记录中心（独立区域）

登录后或点击“个人中心”再展示：
- 邮箱 Magic Link 登录
- 健康档案
- 历史问诊
- 风险变化时间线
- 随访 / 最近结论摘要

这部分应该是“账号工作区”，而不是首页第一屏。

#### 3. 疾病地图 / 公卫情报（独立页面）

疾病地图保留独立页面，但内容要更像情报面板：
- 区域热点
- 症状趋势
- 官方信源摘要
- 最近同步时间
- 透明说明：哪些是实时抓取，哪些是本地/缓存兜底

## 四、这轮具体要做什么

### Wave 1：先解决“看起来还是旧版 / demo”的问题

1. **把最新分支同步到 `main`**
   - 更新 GitHub 首页 README
   - 让 Vercel Production 指向最新版本

2. **重做首页信息架构**
   - 去掉首页大块“档案中心”压屏效果
   - 加清晰的“游客访问 / 登录同步”双入口

3. **弱化技术错误暴露**
   - 不再在首页显示 `Auth session missing!`
   - 不再直接显示“云端读取异常”这类开发态提示
   - 改成产品化语言，例如“当前使用本机模式，可登录后同步”

### Wave 2：把账号区和问诊流拆开

4. **加清晰的登录入口**
   - Header 里增加“登录 / 游客模式”入口
   - 登录后进入个人工作区

5. **问诊页面聚焦单一任务**
   - 让聊天页只负责：症状输入 → 追问 → 结果 → 医院建议
   - 把与账号、历史记录相关的内容移出首屏

### Wave 3：提升可信感与信息质量

6. **清理 demo 感文案**
   - 删除“像在开发中”的表达
   - 统一成真实产品语气

7. **增强官方资料展示**
   - 展示更明确的来源、时间、状态
   - 把“官方信源摘要 + 更新时间 + 获取方式”做得更可信
   - 保留透明说明：实时 / 缓存 / 内置兜底

### Wave 4：收尾上线

8. **同步到 GitHub 与 Vercel**
   - 推送到远程
   - 切到 `main`
   - 触发生产部署
